### Architecture Note

This project uses an **agent-driven architecture**: many library modules (`runner.py`, `experiment_logger.py`, `loop_helpers.py`, `checkpoint.py`) provide utility functions that the Claude Code agent invokes via inline Python per CLAUDE.md instructions, rather than being called through traditional module imports. This is by design — the agent IS the orchestrator.

### Issues Found and Resolved During Audit

**SWARM-07 (Fixed):** `swarm_claude.md.tmpl` was never rendered or written to agent worktrees. Added `render_swarm_claude_md()` to `templates/__init__.py` and write step to `SwarmManager.setup()`. Agents can now read the coordination protocol.

### Accepted Tech Debt

1. **PIPE-02/PIPE-05:** `split_data()` holdout available in `prepare.py` but not called by default `train_template.py`. Agent uses cross-validation on full dataset. Holdout can be wired in v2.
2. **Library modules without production callers:** `ExperimentRunner`, `ExperimentLogger`, `loop_helpers` are tested utilities the agent invokes via CLAUDE.md instructions rather than Python imports. This is the intended agent-driven architecture.
3. **RES-03:** `--resume` flag is informational only (by design per requirement spec).
4. **SWARM-02:** Claim enforcement is agent-driven via `swarm_claude.md`, not orchestrator-enforced.
