## Critical Integration Gaps

### P0: Engine Subprocess — System Cannot Run Experiments

The `RunEngine._run_one_experiment()` spawns `claude -p` with two invalid flags:

1. **`--append-system-prompt-file CLAUDE.md`** (engine.py:129) — Not a valid claude CLI flag. The correct flag is `--append-system-prompt` which takes an inline string argument.
2. **`--max-turns N`** (engine.py:127) — Not a valid claude CLI flag.

**Impact:** Every experiment subprocess fails immediately. The deviation handler treats all results as crashes, reverting continuously. No experiment can actually run.

**Affected requirements:** CORE-02, CORE-03, and transitively all runtime requirements.

### P1: Phase 5 Features Unreachable

1. **DL/FT plugins not registered** — `scaffold.py` only auto-registers `TabularPlugin`. Running `--domain deeplearning` or `--domain finetuning` raises `KeyError` at `get_plugin()`. All DL-01–05 and FT-01–05 are unreachable.

2. **Swarm mode has no CLI entry point** — No `--swarm` or `--n-agents` flag in `cli.py`. `SwarmManager` cannot be invoked. Additionally, `verify_best_result()` is never called even within `SwarmManager.run()`.

### P2: Intelligence Subsystem Orphaned

Six intelligence modules exist, are tested, and pass phase verification — but none are called at runtime:

| Module | Functions | Requirement | Runtime Callers |
|--------|-----------|-------------|-----------------|
| `intelligence/diagnostics.py` | `diagnose_regression`, `diagnose_classification` | INTL-03 | None |
| `intelligence/stagnation.py` | `check_stagnation`, `trigger_stagnation_branch` | INTL-04 | None |
| `intelligence/drafts.py` | `select_best_draft`, `ALGORITHM_FAMILIES` | INTL-05 | None |
| `journal.py` | `append_journal_entry`, `get_last_diff` | CORE-08, INTL-06 | None |
| `tabular/baselines.py` | `compute_baselines`, `passes_baseline_gate` | INTL-01, INTL-02 | None |

These are enforced only as CLAUDE.md text rules, not programmatic gates.

### P3: Simple Mode Incomplete

`cli.py` calls `profile_dataset()` and uses `metric` and `direction`, but does not propagate `task`, `csv_path`, or `target_column` to `plugin_settings`. `TabularPlugin.scaffold()` renders `train.py` with hardcoded defaults.
