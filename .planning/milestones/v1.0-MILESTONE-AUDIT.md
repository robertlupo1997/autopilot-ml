## Integration Gaps

### GAP-1: Tabular train.py outputs plain text, engine expects JSON (P0)

**Affected:** CORE-02, INTL-01, INTL-02, INTL-03, TABL-01, TABL-04

`tabular_train.py.j2` outputs `print(f"Best value: {study.best_value:.4f}")` (plain text) but `engine._process_result()` extracts `metric_value` from `json.loads(result["result"])`. The DL and FT templates correctly use `json.dumps()` -- only tabular is missing this.

**Impact:** Every tabular experiment gets `metric_value=None`, causing all experiments to revert. The entire tabular experiment loop is non-functional at runtime.

### GAP-2: CLAUDE.md has no output format instruction (P1)

**Affected:** CORE-03, TABL-04

`base_claude.md.j2` has no section telling the agent to emit `{"metric_value": X}` in its output. Even if train.py produced the right output, the agent protocol doesn't instruct the agent to relay it.

### GAP-3: Stagnation crashes when no keep has occurred (P1)

**Affected:** CORE-09, INTL-04

`trigger_stagnation_branch()` raises `ValueError` when `state.best_commit is None`. Engine does not guard against this. Common failure path when all experiments revert.

### GAP-4: validate_config() never called (P2)

**Affected:** TABL-01, DL-01, FT-01, FT-04

All three plugins define `validate_config()` but it is never called from scaffold.py, cli.py, or engine.py. Invalid configs proceed silently. `FineTuningPlugin` specifically requires `model_name` -- not enforced.

### GAP-5: DL/FT simple mode gets wrong task type (P2)

**Affected:** DL-01, DL-03, UX-01

Profiler outputs `"classification"/"regression"` but DL plugin expects `"image_classification"/"text_classification"`. Simple mode renders wrong model architecture for DL/FT domains.

### GAP-6: tag_best() never called (P3)

**Affected:** CORE-10

`GitManager.tag_best()` defined in git_ops.py but never invoked from RunEngine. CORE-10 requires "tag best model" -- tagging is dead code.

### GAP-7: Swarm scoreboard population is protocol-only (P3)

**Affected:** SWARM-01, SWARM-02

`SwarmScoreboard.publish_result()` exists but is never called from code. Agents are text-prompted to write TSV rows themselves. If the `claude -p` agent doesn't comply, the scoreboard stays empty and verification returns None.

### GAP-8: DatasetProfile rich fields discarded (P3)

**Affected:** UX-04

`profile_dataset()` computes `missing_pct`, `numeric_features`, `categorical_features`, `leakage_warnings` but `cli.py` only uses `task`, `metric`, `direction`, `date_columns`. The rich profile data goes nowhere.
