Write(train.py) was already present. Write(*) would be broader than needed -- the hook on Edit|Write enforces frozen file protection, but defense-in-depth says keep Write scoped.

   **Why NOT scoped Bash rules like `Bash(uv run python *)` and `Bash(git *)`:** The FINDINGS.md proposed scoped rules, but `Bash(*)` is simpler and the Phase 4 baseline proved the agent only runs appropriate commands (uv, git, python). The hook system protects files via Edit|Write, not Bash. Bash(*) matches the intent of the original bare "Bash" rule but works reliably in headless mode.
