  <action>
Add `_run_draft_phase(self) -> list[DraftResult]` method to RunEngine:
- Iterate over `ALGORITHM_FAMILIES.items()`
- For each family: determine task from `self.config.plugin_settings.get("task", "classification")`
- Build a draft-specific prompt via `_build_draft_prompt(family_name, family_info, task)` that instructs the agent to use a specific model class
- Call `_run_one_experiment()` with the draft prompt (modify `_run_one_experiment` to accept an optional `prompt_override: str | None = None` parameter -- if provided, use it instead of `_build_prompt()`)
- Extract metric_value from the result (same nested JSON parsing as `_process_result`)
- If result has metric_value, commit as `f"draft-{family_name}"` and create `DraftResult(name=family_name, metric_value=..., status="draft-keep", commit_hash=..., description=...)`
- If result has no metric_value (crash/timeout), create `DraftResult(name=family_name, metric_value=None, status="draft-discard", commit_hash="", description=...)`
- Track cost via `self.cost_tracker.record()`
- Append family_name to `self.state.tried_families`
- Return all DraftResult objects

- Build a prompt like: "You are an ML research agent. Read CLAUDE.md for your protocol. This is a DRAFT experiment. Use ONLY {family_info[task]} from {family_info['description']}. Do NOT use other model families. Run train.py, evaluate results, and report the metric value. Metric: {self.config.metric} (direction: {self.config.direction})."
