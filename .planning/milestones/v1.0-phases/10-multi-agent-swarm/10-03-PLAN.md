    - WARNING: "Must be run from a terminal OUTSIDE of Claude Code"

    - What it does: scaffolds experiment from noisy.csv, runs 2-agent swarm, checks scoreboard

    Script body:
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail

    SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
    TEST_DIR=$(mktemp -d)
    FIXTURE="$PROJECT_ROOT/tests/fixtures/noisy.csv"

    echo "=== Multi-Agent Swarm Smoke Test ==="
    echo "Test directory: $TEST_DIR"
    echo ""

    # 1. Scaffold experiment
    echo "[1/4] Scaffolding experiment..."
    cd "$PROJECT_ROOT"
    uv run automl "$FIXTURE" target accuracy \
        --goal "Swarm smoke test" \
        --output-dir "$TEST_DIR/experiment-swarm" \
        --time-budget 60

    # 2. Run 2-agent swarm (low max-turns for smoke test)
    echo "[2/4] Running 2-agent swarm (this will take a few minutes)..."
    cd "$TEST_DIR/experiment-swarm"
    # Install dependencies
    uv sync 2>/dev/null || true

    # Spawn swarm via CLI (the CLI handles SwarmManager internally)
    # For smoke test: use --agents 2, each agent gets ~25 turns
    uv run automl "$FIXTURE" target accuracy \
        --goal "Swarm smoke test" \
        --output-dir "$TEST_DIR/experiment-swarm-run" \
        --time-budget 60 \
        --agents 2 || true
    # Note: above may fail since experiment dir already exists
    # Alternative: invoke SwarmManager directly from the scaffolded dir

    # 3. Check results
    echo "[3/4] Checking results..."
    SWARM_DIR="$TEST_DIR/experiment-swarm/.swarm"
    if [ -f "$SWARM_DIR/scoreboard.tsv" ]; then
        ROWS=$(tail -n +2 "$SWARM_DIR/scoreboard.tsv" | wc -l)
        echo "  Scoreboard rows: $ROWS"
        echo "  Contents:"
        cat "$SWARM_DIR/scoreboard.tsv"
    else
        echo "  WARNING: scoreboard.tsv not found"
    fi

    # 4. Cleanup
    echo "[4/4] Cleaning up..."
    rm -rf "$TEST_DIR"

    echo ""
    echo "=== Swarm Smoke Test Complete ==="
    ```

    Make executable: chmod 755

    IMPORTANT: The exact invocation pattern may need adjustment based on how Plan 10-02 wires the CLI --agents flag to SwarmManager. The script should use whatever entry point Plan 10-02 establishes. If the CLI creates a new experiment dir (scaffold) before running swarm, the script should do scaffold first then swarm. If the CLI does both in one call, adjust accordingly.

    The script is deliberately simple -- a smoke test to verify the plumbing works. Real swarm validation requires API credits and time, so this script is run manually by the user.
