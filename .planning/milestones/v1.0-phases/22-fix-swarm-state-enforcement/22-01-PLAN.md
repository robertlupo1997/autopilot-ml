<objective>
Make swarm agent result collection code-enforced instead of relying solely on AI text instruction compliance.

Purpose: Currently SwarmManager.run() reads state.json from each agent worktree after completion, but state.json is only written if the AI agent follows a text instruction in swarm_claude.md.j2. If the agent skips, crashes, or misformats the write, results are silently lost (pass in except block). This plan adds subprocess stdout capture as the primary result source, checkpoint.json as fallback, and replaces silent failure with explicit handling.

Output: Updated swarm/__init__.py with robust result collection + new test file
</objective>
