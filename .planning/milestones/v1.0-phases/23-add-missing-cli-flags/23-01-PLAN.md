# Plan: Add --model-name and --direction CLI Flags

## Goal
Close INT-FT-MODEL-NAME and INT-DIRECTION-FLAG integration gaps from v1.0 audit by adding two CLI arguments.

## Files to Modify
- `src/mlforge/cli.py` — add `--model-name` and `--direction` argparse arguments, wire into config/plugin_settings

## Implementation

### Task 1: Add --model-name argument
Add after the `--enable-drafts` argument:
```python
parser.add_argument("--model-name", type=str, default=None, help="HuggingFace model name for fine-tuning domain")
```

### Task 2: Add --direction argument
```python
parser.add_argument("--direction", choices=["minimize", "maximize"], default=None, help="Metric optimization direction override")
```

### Task 3: Wire --model-name to plugin_settings
In the CLI run flow, after config setup:
```python
if args.model_name is not None:
    config.plugin_settings["model_name"] = args.model_name
```

### Task 4: Wire --direction to config
```python
if args.direction is not None:
    config.direction = args.direction
```

## Tests
- Test `--model-name` is accepted by argparse and appears in plugin_settings
- Test `--direction minimize` sets config.direction to "minimize"
- Test `--direction maximize` sets config.direction to "maximize"
- Test default behavior unchanged when flags not provided

## Verification
- `mlforge dataset goal --domain finetuning --model-name meta-llama/Llama-3.2-1B` works without argparse error
- `mlforge dataset goal --metric rmse --direction minimize` correctly sets direction=minimize
