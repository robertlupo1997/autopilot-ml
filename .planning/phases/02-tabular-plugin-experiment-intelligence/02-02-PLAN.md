  <name>Task 1: Diagnostics engine for regression and classification</name>
  <files>
    src/mlforge/intelligence/__init__.py,

  <behavior>
    - diagnose_regression(y_true, y_pred, feature_names, X, top_n=5) returns dict with worst_predictions, bias, feature_error_correlations
    - worst_predictions is list of top_n dicts with index, y_true, y_pred, abs_error sorted by abs_error descending
    - bias has direction ("over"/"under"/"neutral") and magnitude (float)
    - feature_error_correlations maps feature_name to correlation coefficient (only when X and feature_names provided)
    - features with zero std are excluded from correlations
    - diagnose_classification(y_true, y_pred, class_names=None, y_proba=None, top_n=5) returns dict with misclassified_samples, per_class_accuracy, confused_pairs
    - misclassified_samples lists samples where prediction was wrong, sorted by confidence (if y_proba provided)
    - per_class_accuracy maps class label to accuracy float
    - confused_pairs lists (true_class, predicted_class, count) tuples sorted by count descending
  </behavior>
  <action>
    1. Create src/mlforge/intelligence/__init__.py (empty, makes it a package).

    2. Create src/mlforge/intelligence/diagnostics.py with two functions:

    diagnose_regression(y_true, y_pred, feature_names=None, X=None, top_n=5) -> dict:
    - Compute error = y_pred - y_true, abs_error = np.abs(error)
    - worst_predictions: argsort abs_error descending, take top_n, return list of {index, y_true, y_pred, abs_error}
    - bias: mean_error > 0 = "over", < 0 = "under", == 0 = "neutral", magnitude = mean_error
    - feature_error_correlations: if X and feature_names provided, compute np.corrcoef(X[:, j], abs_error)[0, 1] for each feature where std > 0
    - Adapted from old forecast.diagnose() pattern (see research doc)

    diagnose_classification(y_true, y_pred, class_names=None, y_proba=None, top_n=5) -> dict:
    - misclassified_samples: indices where y_true != y_pred, if y_proba provided sort by max predicted probability (highest confidence wrong predictions first), take top_n
    - per_class_accuracy: for each unique class, compute accuracy = correct/total for that class
    - confused_pairs: count (true, pred) pairs where true != pred, sort by count descending, take top_n

    3. Write tests FIRST in tests/mlforge/test_diagnostics.py:
    - test_regression_worst_predictions: synthetic data with known worst errors
    - test_regression_bias_over/under/neutral: synthetic data with known bias direction
    - test_regression_feature_correlations: one feature perfectly correlated with error
    - test_regression_zero_std_feature: feature with zero std excluded
    - test_classification_misclassified: known misclassifications detected
    - test_classification_per_class_accuracy: computed correctly per class
    - test_classification_confused_pairs: correct pairs and ordering
    - test_classification_with_proba: confidence-sorted misclassifications

    Use numpy arrays directly (no sklearn needed for diagnostics tests).
  </action>
  <verify>
    <automated>python3 -m pytest tests/mlforge/test_diagnostics.py -x -v</automated>
  </verify>
  <done>
    - diagnose_regression returns correct worst predictions, bias, and correlations for synthetic data
    - diagnose_classification returns correct misclassifications, per-class accuracy, and confused pairs
    - Edge cases handled (zero std features, no proba, neutral bias)
    - All tests pass
  </done>

    - DraftResult dataclass has: name, metric_value (float|None), status (str), commit_hash (str), description (str)
    - select_best_draft(results, direction="maximize") returns DraftResult with best metric
    - select_best_draft returns max for maximize, min for minimize
    - select_best_draft returns None when no results have metric_value
    - select_best_draft filters out results with metric_value=None
    - check_stagnation(state, threshold=3) returns True when consecutive_reverts >= threshold
    - check_stagnation returns False when consecutive_reverts < threshold
    - trigger_stagnation_branch(git_manager, state, new_family) creates branch "explore-{new_family}" from best_commit
    - trigger_stagnation_branch resets state.consecutive_reverts to 0
    - trigger_stagnation_branch raises ValueError if state.best_commit is None
  </behavior>
  <action>
    1. Create src/mlforge/intelligence/drafts.py:
    - ALGORITHM_FAMILIES dict with entries for: "linear" (LogisticRegression/Ridge), "random_forest" (RandomForestClassifier/Regressor), "xgboost" (XGBClassifier/Regressor), "lightgbm" (LGBMClassifier/Regressor), "svm" (SVC/SVR)
    - Each entry: {"description": str, "classification": str (class name), "regression": str (class name)}
    - DraftResult dataclass: name, metric_value (float|None), status (str, "draft-keep"|"draft-discard"), commit_hash (str), description (str)
    - select_best_draft(results: list[DraftResult], direction: str = "maximize") -> DraftResult | None: filter valid (metric_value not None), return max/min by direction, None if no valid results

    2. Create src/mlforge/intelligence/stagnation.py:
    - check_stagnation(state: SessionState, threshold: int = 3) -> bool: return state.consecutive_reverts >= threshold
    - trigger_stagnation_branch(git_manager: GitManager, state: SessionState, new_family: str) -> str:
      - Raise ValueError if state.best_commit is None
      - git_manager.repo.git.checkout(state.best_commit)
      - branch_name = f"explore-{new_family}"
      - git_manager.repo.create_head(branch_name).checkout()
      - state.consecutive_reverts = 0
      - return branch_name
