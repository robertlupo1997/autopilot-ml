<task type="auto" tdd="true">
  <name>Task 1: ResourceGuardrails + CostTracker + DeviationHandler + SessionState.cost_spent_usd</name>
  <files>src/mlforge/guardrails.py, src/mlforge/state.py, tests/mlforge/test_guardrails.py</files>
  <behavior>
    ResourceGuardrails:
    - should_stop returns True when experiment_count >= budget_experiments
    - should_stop returns True when cost_spent_usd >= budget_usd
    - should_stop returns True when elapsed time >= budget_minutes * 60
    - should_stop returns True when free disk < min_free_disk_gb (default 1.0)
    - should_stop returns False when all budgets have room
    - stop_reason returns human-readable string explaining which guardrail tripped

    CostTracker:
    - record(cost_usd) adds to running total and updates SessionState.cost_spent_usd
    - total_cost returns accumulated cost
    - per_experiment_costs returns list of individual experiment costs

    DeviationHandler:
    - handle returns "retry" when result status is "crash" and error contains "MemoryError" or "OOM", and retries < MAX_RETRIES (2)
    - handle returns "revert" when result status is "crash" (non-OOM)
    - handle returns "revert" when result status is "timeout"
    - handle returns "revert" when metric_value is None, NaN, or inf
    - handle returns "revert" when metric does not improve over best (respecting direction)
    - handle returns "keep" when metric improves over best (respecting direction)
    - handle returns "stop" when retries exhausted on OOM

    SessionState:
    - cost_spent_usd field defaults to 0.0
    - cost_spent_usd round-trips through JSON (to_json/from_json)
  </behavior>
  <action>
    1. Add `cost_spent_usd: float = 0.0` field to SessionState in state.py. No other changes to state.py needed (existing to_json/from_json use asdict/fields which auto-include new fields).

    2. Create guardrails.py with three classes:

    **ResourceGuardrails(config: Config, experiment_dir: Path):**
    - `__init__` records start_time = time.time(), stores config and experiment_dir, sets min_free_disk_gb = 1.0
    - `should_stop(state: SessionState) -> bool` checks four conditions
    - `stop_reason(state: SessionState) -> str | None` returns which guardrail tripped (or None)
    - Uses `shutil.disk_usage(self.experiment_dir)` for disk check

    **CostTracker:**
    - `__init__` initializes _costs: list[float] = [], _total = 0.0
    - `record(cost_usd: float, state: SessionState)` appends cost and updates state.cost_spent_usd
    - `total_cost -> float` property
    - `per_experiment_costs -> list[float]` property

    **DeviationHandler(direction: str = "maximize"):**
    - `MAX_RETRIES = 2`
    - `__init__` tracks _retry_count = 0
    - `handle(result: dict, state: SessionState) -> str` returns one of "keep", "revert", "retry", "stop"
    - On successful keep: reset _retry_count to 0
    - On OOM retry: increment _retry_count, if >= MAX_RETRIES return "stop"
    - Improvement check: for "maximize" new > best, for "minimize" new < best. If best_metric is None, any finite metric is an improvement.
    - Uses `math.isfinite()` for NaN/inf detection

    3. Write tests first. Use monkeypatch for time.time() and shutil.disk_usage(). Test each condition independently.
  </action>
  <verify>
    <automated>python -m pytest tests/mlforge/test_guardrails.py -x -q</automated>
  </verify>
  <done>All guardrail conditions tested and working. Cost tracker accumulates correctly. Deviation handler routes crash/OOM/timeout/divergence/improvement correctly. SessionState persists cost_spent_usd.</done>
</task>
