## Root Cause Analysis

The `settings.json` file generated by the scaffold (via `src/automl/scaffold.py`) has permissions.allow rules that are too narrow for headless operation. The likely current allow rules are scoped to `Edit(train.py)` and `Write(train.py)`, but the autonomous loop also needs:

1. **Bash** tool: To run experiments (`uv run python train.py`), check packages, run git commands
2. **Write** tool: To create/update `results.tsv`, `run.log`, and other output files
3. **Edit** tool: To modify `train.py` during draft and iteration phases

In interactive mode, the user approves these prompts on the fly. In headless `claude -p` mode, unapproved tool calls are denied without recourse. The fix is to broaden the permissions.allow rules in the generated settings.json.

## Fix Required Before v1.0

The scaffold must generate a `settings.json` with permissions sufficient for fully unattended operation. Proposed allow list:

```json
{
  "permissions": {
    "allow": [
      "Bash(uv run python *)",
      "Bash(git *)",
      "Bash(cat *)",
      "Bash(grep *)",
      "Write(results.tsv)",
      "Write(run.log)",
      "Edit(train.py)"
    ],
    "deny": [
      "Edit(prepare.py)",
      "Write(prepare.py)"
    ]
  }
}
```

Alternatively, use broader glob patterns: `Bash(*)`/`Write(*)`/`Edit(train.py)` — accepting that hooks enforce the deny-list for frozen files.
