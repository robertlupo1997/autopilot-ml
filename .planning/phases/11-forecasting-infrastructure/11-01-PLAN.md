<objective>
Create the `forecast.py` frozen module with walk-forward temporal evaluation, forecasting metrics (MAPE, MAE, RMSE, directional accuracy), and naive/seasonal-naive baselines. Update the guard hook to protect the new module immediately. This is the core leakage-free infrastructure that all v2.0 forecasting experiments depend on.

Purpose: The agent must never be able to break temporal ordering during evaluation. By freezing walk-forward logic in `forecast.py`, the infrastructure guarantees all test-fold indices are strictly after all train-fold indices. This is the single most important correctness property for time-series ML.

Output: `src/automl/forecast.py` (production module), `tests/test_forecast.py` (comprehensive tests), updated `tests/conftest.py` (shared fixture), updated `src/automl/scaffold.py` (guard hook protection).
