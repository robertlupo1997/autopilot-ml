<objective>
Add best-commit tracking to experiments.md and replace the existing stagnation-recovery protocol in both CLAUDE.md templates with branch-on-stagnation: after 3 consecutive reverts, the agent branches back to its best-ever commit and tries a different model family.

Purpose: Instead of blindly switching strategy from a potentially degraded state, the agent returns to its proven best result and explores from there — capturing the core value of tree search (backtracking to the best node) without MCTS complexity.

Output: Updated experiments.md.tmpl with Best Result section, updated claude.md.tmpl and claude_forecast.md.tmpl with exploration branch protocol, structural tests confirming all EXPL requirements.
</objective>

       d. Continue the iteration loop from step 1 on the exploration branch.

       e. Reset the consecutive revert counter after branching.
