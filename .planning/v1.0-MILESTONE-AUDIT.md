## Cross-Phase Integration

**Score: 42/45 connections wired (3 issues found)**

### Connected (42 exports)

All phase exports have at least one downstream consumer. Core modules (state, config, checkpoint, git_ops, journal, plugins, templates, hooks) are consumed by every downstream phase. Intelligence modules are imported and called in engine.py. Plugin registration dispatch works for all 3 domains. Swarm coordination is reachable via CLI.

### Integration Issues

**1. HIGH — Baseline gate is dead code in runtime (INTL-01, INTL-02)**

`_compute_baselines()` in `engine.py` executes `prepare.py` via importlib and looks for `X_train`/`y_train` as module-level attributes. But `prepare.py` is a pure-functions library — it defines `load_data()`, `split_data()`, etc. but never assigns data at module scope. Result: `state.baselines` is always `None`, the dual-baseline gate never fires.

**Fix:** Have `_compute_baselines()` call `prepare.load_data()` + `prepare.split_data()` using the dataset path from `config.plugin_settings`.

**2. MEDIUM — Multi-draft unreachable via CLI (INTL-05)**

`enable_drafts` defaults to `False` in Config with no CLI flag to enable it. The multi-draft phase code exists and is tested, but users can't activate it without manually editing TOML config.

**Fix:** Add `--enable-drafts` CLI flag in `cli.py`.

**3. LOW — Swarm --cwd flag is invalid (SWARM-01)**

`_build_agent_command()` passes `--cwd` to the `claude` CLI, which is not a recognized flag. Agents still work because `Popen(cwd=...)` sets the directory correctly at the OS level.

**Fix:** Remove `--cwd` from the command list in `swarm/__init__.py`.
